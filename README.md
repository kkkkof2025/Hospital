# 采血管的缤纷之旅（检验医学游戏）

## 项目说明
这是一个纯前端、可直接在浏览器运行的小游戏：在 30 秒记忆阶段记住采血管颜色与项目配对，进入 30 秒挑战阶段后，把掉落的项目卡片拖拽到正确的采血管中，获得分数并得到评价。

## 开发过程记录
1. 需求拆解
- 整理场景流程：开始画面 -> 记忆阶段（30 秒、每张卡片 2 秒） -> 游戏阶段（30 秒） -> 结算评分。
- 明确核心交互：卡片从上方掉落，玩家拖拽卡片到正确颜色采血管。
- 效果需求：计时条、得分、成功/失败提示、音效、背景检验科氛围。

2. 视觉与布局设计
- 采用卡通实验室风格：渐变背景 + 软性阴影 + 设备剪影。
- 角色“赛斯”使用纯 CSS 形状绘制，并加入眨眼与挥手动画。
- 底部 6 支采血管并排，顶部显示计时与得分。

3. 数据结构与逻辑设计
- 用 `CARD_DATA` 维护颜色、项目与图标的对应关系。
- 记忆阶段：每 2 秒切换卡片，并用统一的 30 秒计时条控制结束。
- 游戏阶段：循环生成卡片，随时间加速下落。
- 拖拽交互：使用 Pointer Events，检测卡片释放点是否落在正确管子范围内。

4. 功能实现
- 开始前 3、2、1 倒计时覆盖层。
- 记忆阶段计时条与卡片循环展示。
- 游戏阶段生成卡片、拖拽配对、加/减分与反馈闪烁。
- 结束阶段输出得分与评价，支持重新挑战或返回开始。
- 使用 Web Audio API 生成轻量成功/失败提示音。

5. 联调与测试
- 在桌面浏览器测试拖拽与计时逻辑。
- 在小屏宽度下验证布局响应式（卡片大小与采血管布局自适应）。

## 文件结构
- `index.html`：页面结构与三段流程界面。
- `styles.css`：整体视觉、背景、角色、采血管、卡片与动画样式。
- `game.js`：流程控制、卡片生成/拖拽、计时与计分逻辑。

## 本地运行
1. 直接双击打开 `index.html`。
2. 或使用任意静态服务器（如 VSCode Live Server）打开即可。

## 玩法说明
- 记忆阶段 30 秒：每张卡片停留 2 秒，请记住“颜色 - 项目 - 图标”。
- 游戏阶段 30 秒：拖拽掉落卡片到对应颜色采血管。
- 配对正确：+10 分；配对错误：-5 分。

## 可扩展方向（可选）
- 加入更多检验项目与管色组合。
- 增加关卡或难度（更多颜色、更快速度）。
- 增加背景音效或角色语音。
